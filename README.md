## 🧠 How I Learn

### 1. Anki Review

Before I start coding, I review concepts from previous days using Anki.

The goal isn't recognition.

The goal is recall.

If I can't explain something from memory, I don't really know it yet.

### 2. Define the Build Goal

Before opening my editor, I write one sentence:

> I will build X using Y. Done means Z works.

If I can't define the task clearly, I'm not ready to start.

### 3. The 25-Minute Rule

When I hit a bug, I spend at least 25 minutes trying to solve it myself before searching for help.

Some bugs disappear.

Others teach me how to think.

Every meaningful bug gets documented with:

* What broke
* Why it broke
* How I fixed it
* What I learned

### 4. Feynman Reflection

At the end of each session, I answer three questions:

1. What did I build?
2. How does it work?
3. What don't I fully understand yet?

If I can't explain something simply, I still have work to do.
